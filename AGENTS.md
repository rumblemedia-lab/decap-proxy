# AGENTS.md

A Cloudflare Worker that acts as a GitHub OAuth proxy for [Decap CMS](https://github.com/decaporg/decap-cms). It lets Decap's browser admin panel authenticate against GitHub without Netlify Identity / Git Gateway. The worker is intentionally tiny: two source files, two routes, no runtime dependencies.

## Commands

Package manager is **yarn** (Node 24, see `.nvmrc`). There is no build step — Wrangler bundles `src/index.ts` directly.

```bash
yarn dev          # local dev server via wrangler dev (also: yarn start)
yarn deploy       # wrangler deploy
yarn test         # run vitest suite
yarn test:debug   # same, with --run --disableConsoleIntercept for readable output
yarn cf-typegen   # regenerate worker-configuration.d.ts from wrangler.toml bindings
```

## Critical gotcha: `wrangler.toml` is gitignored

The repo ships `wrangler.toml.sample` only. **`wrangler.toml` must exist before `yarn dev`, `yarn deploy`, or `yarn test` will work** — the vitest config (`vitest.config.ts`) points Wrangler at `./wrangler.toml`, and `wrangler dev`/`deploy` require it too. CI handles this with `mv wrangler.toml.sample wrangler.toml` (see `.github/workflows/test.yml`). When working locally, copy the sample first:

```bash
cp wrangler.toml.sample wrangler.toml
```

`GITHUB_OAUTH_ID` and `GITHUB_OAUTH_SECRET` are **secrets**, not vars — they are not in `wrangler.toml`. Set them via `npx wrangler secret put ...` or the Cloudflare dashboard. Tests do not require them (the existing test snapshot even asserts `client_id=undefined`).

`GITHUB_REPO_PRIVATE` **is** a plaintext `[vars]` entry in `wrangler.toml` (`"0"` by default). It controls the OAuth scope: `"1"` → `repo,user`, otherwise `public_repo,user`.

## Architecture

**`src/index.ts`** — Worker entry. Default export `{ fetch }` routes by `url.pathname`:

- `/auth` → `handleAuth`: 301 redirect to GitHub's authorize URL. Scope chosen from `GITHUB_REPO_PRIVATE`. `state` is a fresh 4-byte hex string from the Web Crypto API (`crypto.getRandomValues`). Redirect URI is built from `url.hostname`, so the proxy works on both `workers.dev` and custom domains without extra config.
- `/callback` → `handleCallback`: receives GitHub's `code`, POSTs to `/login/oauth/access_token` to exchange it for an access token, then returns `callbackScriptResponse(...)`.
- anything else → `new Response('Hello 👋')` (this is the deployment smoke test referenced in the README).

**`callbackScriptResponse`** returns a small HTML page whose inline script uses `window.opener.postMessage` with the exact protocol Decap CMS expects: `'authorization:github:success:<token>'` (after first sending `'authorizing:github'` and waiting for a message back). If you change the callback flow, you must preserve this `postMessage` contract or Decap's auth will silently break.

**`src/oauth.ts`** — `OAuthClient` class (plain TS, no OAuth library dependency). `authorizeURL()` builds the GitHub authorize URL by string concatenation; `getToken()` POSTs a JSON body (with `Accept: application/json`) and returns `json.access_token`. Note `getToken` does not check `response.ok` or handle GitHub errors — a non-JSON or error response will throw.

The worker uses only the Web Crypto API and native `fetch`; **no `nodejs_compat` flag is needed** (this was a deliberate change to avoid Node polyfill vulnerabilities — see commit `d19020b`).

## Testing

Tests run **inside the Workers runtime**, not Node, via `@cloudflare/vitest-pool-workers` (`vitest.config.ts` uses `defineWorkersConfig`). This is why `wrangler.toml` must exist (see gotcha above).

Key patterns from `test/index.spec.ts`:

- Import from `cloudflare:test` (not `vitest`) for `SELF` and `fetchMock`.
- `SELF.fetch('https://example.com/...')` invokes the worker as if it were deployed; hostnames in the test URL become the worker's hostname.
- `beforeAll`: `fetchMock.activate()` then `fetchMock.disableNetConnect()` — all network is mocked.
- Mock GitHub by chaining: `fetchMock.get('https://github.com').intercept({ path, method }).reply(status, body)`.
- `cloudflare:test` types come from `@cloudflare/vitest-pool-workers`, declared in `test/tsconfig.json`.

There is **no separate lint command** — formatting is enforced only by the `.prettierrc` / `.editorconfig` config; check the style section below.

## TypeScript layout (two tsconfigs)

- Root `tsconfig.json` — for source. `strict: true`, `noEmit: true`, `target`/`lib` `es2021`, `module` `es2022`, `moduleResolution` `Bundler`. **Excludes `test`.** Types include `@cloudflare/workers-types/2023-07-01` and `./worker-configuration.d.ts`.
- `test/tsconfig.json` — extends root, **includes tests**, swaps the types to `@cloudflare/workers-types/experimental` + `@cloudflare/vitest-pool-workers`, and lists `../src/env.d.ts` in `include`. Note: `src/env.d.ts` does **not** currently exist in the repo, and `worker-configuration.d.ts` (generated by `yarn cf-typegen`) currently declares an empty `interface Env {}`. The real `Env` interface is hand-written inline in `src/index.ts`. If you add worker bindings, run `yarn cf-typegen` and be aware the inline `Env` in `index.ts` may need to stay in sync.

## Code style

- **Indentation is tabs**, not spaces (`.editorconfig` + `.prettierrc` `useTabs: true`). This is non-negotiable and easy to get wrong if an editor defaults to spaces.
- Single quotes, semicolons, `printWidth: 140`.
- Exception: **`*.yml` uses spaces** (`.editorconfig` override) — relevant when editing `wrangler.toml.sample`-adjacent YAML or workflow files.
- LF line endings, trim trailing whitespace, final newline required.

## Commit / PR conventions

From `git log`: commit messages are short, lowercase, imperative, no conventional-commit prefixes (`add ci test workflow`, `fix mv file name`, `Use native Web Crypto API...`). Dependency bumps land via Dependabot PRs titled `Bump <pkg> from X to Y (#NN)`. PRs are squash-merged.
